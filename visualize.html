d3.js 소개.

웹 브라우저 상에서 작동하는(자바스크립트) 데이터 시각화용 라이브러리.
SVG, HTML5, CSS 표준 기술을 사용해 구현됨.

Protovis라는 framework가 현재 d3의 전신이라고 함.
초기 버전은 2011년에 출시되서 현재까지 계속 배포되고 있다고 함.
정적인 컨텐츠들만 보여주던 기존의 웹의 한계를 뛰어 넘고자, 웹브라우저에 자바스크립트를 쓸 수 있게 통합시키려는 노력이 이루어졌고, 이러한 결정의 결과로 d3가 자바스크립트를 채택하게 되었다고 한다.

Prefuse, Flare, and Protovis toolkits 같은 프로젝트들이 d3의 전신이다.
html 페이지에 내장되서 돌아가는 d3 라이브러리는 Document object, SVG object등을 생성, 변형할 수 있다. 그리고, 그 객체들에 애니메이션을 줄수도 있다!
이런 객체들에 CSS를 통해서 스타일링을 입힐 수 있다.!
다행히도, d3는 다양한 데이터들을 취급할수 있게 설계되어 있다.

가장 많이 사용되는 데이터형태는 JSON, CSV, geoJSON등이 있다.
물론, 원하면 다른 데이터 형태도 읽을 수 있도록, 함수를 만들면 된다.

d3는 코드는 함수형 언어적인 측면이 있다.

왜? 시각화를 하는가?

위키피디아의 정의 : 데이터 시각화(data visualization)는 데이터 분석 결과를 쉽게 이해할 수 있도록 시각적으로 표현하고 전달되는 과정을 말한다.

목적 : 데이터 시각화의 목적은 도표(graph)라는 수단을 통해 정보를 명확하고 효과적으로 전달하는 것이다.

# Selection

W3C(웹 표준 기구)의 DOM API로 object를 조작하는 일들은 지루하다.(메소드 이름도 길고, 하는 일도 명시적이지 않다.)
예를 들어서 다음과 p tag의 텍스트 색상을 바꾸는 일을 해보자.

p-tag-list 고정으로 띄우기

          var paragraphs = document.getElementsByTagName("p");
          for (var i = 0; i < paragraphs.length; i++) {
            var paragraph = paragraphs.item(i);
            paragraph.style.setProperty("color", "white", null);
          }

반복문을 돌면서, 변수에 상태를 묶어놓는 작업들을 한다.
반면에 d3에서는 다음과 같이 한다. 
        
          d3.selectAll("p").style("color", "white");
위의 코드에서 보듯이, d3는 node를 선택하고 조작하는 많은 방법을 제공해준다.

# Dynamic Properties

jQuery, Prototype 같은 라이브러리 처럼, chain method를 통해서 node(Document Object)의 스타일과, 속성들을 변경할 수 있다. 
        
          d3.selectAll("p")
              .data([4, 8, 15, 16, 23, 42])
              .style("font-size", function(d) { return d + "px"; });

위 예제를 보면, computed property(style("font-size", function(d) { return d + "px"; });)는 바인딩된 데이터를 참조한다.
위 예제에서 데이터는 [4, 8, 15, 16, 23, 42] 값들의 배열로 표현된다.
각각의 값들은 selection function(function(d) { return d + "px"; })의 첫번째 인자(d)로 넘어간다.
한번 데이터가 document에 바인딩 되면, data(.data([list])) 연산을 하지 않아도 된다.
D3는 이전에 바인딩된 데이터를 탐색 후 메꾼다.(retrieve) 이러한 행위는 다시-바인딩 시키는 작업 없이 속성들을 다시 계산하게 해준다.

# Enter and Exit

d3에서 enter, exit 을 통해서 새로 들어오는 데이터들에 대해서 새로운 노드를 생성하거나, 더 이상 필요 없는 노드들을 지울수 있다.

예제를 보자. 
        
          d3.select("body").selectAll("p")
            .data([4, 8, 15, 16, 23, 42])
            .enter().append("p")
            .text(function(d) { return "I’m number " + d + "!"; });

위의 객체들을 하나씩 콘솔에 찍어보면 다음과 같은 결과들이 나온다. 반영된 결과물들은 위의 p-tag-list 를 참조하면서 보면된다.


d3.select("body")
| body 객체가 선택이 된다.

d3.select("body").selectAll("p")
| body 객체 안의 모든 p 객체가 선택이 된다.

d3.select("body").selectAll("p").data([4, 8, 15, 16, 23, 42])
| 선택된 모든 p 객체에 data-set을 binding한다.

d3.select("body").selectAll("p").data([4, 8, 15, 16, 23, 42]).enter()
| d3 selection에 바인드된 데이터들 중 아직 실제 문소 요소를 가지지 못하는 것들을 찾아내서 가상의객체로 만들어 반환해준다.
| <공식 홈페이지 설명> 만약에 노드의 개수(p tag)가 데이터의 개수([4, 8, 15, 16, 23, 42])보다 적으면,
| 여분의 데이터 요소들은([16,23,42]) enter 메소드에서 초기화 되면서, 메모리상에서만 머물게 되고, 화면에는 보이지 않게 된다.

d3.select("body").selectAll("p").data([4, 8, 15, 16, 23, 42]).enter().append("p")
| 가상의 객체를 p객체로 만든다.

d3.select("body").selectAll("p").data([4, 8, 15, 16, 23, 42]).enter().append("p").text(function(d) { return "새로운 p tag 입니다.!!!!"; });
| p 객체 안에 text들을 삽입한다.
| 적용된 결과는 문서 끝에 삽입된다.
